I work on embodied AI systems that use external tools to perceive, reason,
validate, and act in physical or simulated environments. My recent work focuses
on turning heterogeneous perception, planning, and control modules into
callable tools for VLM-driven embodied agents.

## Current Focus

- Tool-augmented embodied intelligence and VLM tool use
- Embodied tool discovery, registration, and validation pipelines
- Robot manipulation, navigation, and physical interaction benchmarks
- Research project pages, documentation, and evaluation interfaces

## Selected Work

### EmbodiedTool

I contribute to EmbodiedTool, a collaborative research codebase for extensible
embodied capabilities with tools. The project organizes perception, cognition,
reasoning, and execution abilities as independently callable modules for
embodied agents, with evaluation coverage across simulation benchmarks and
physical robot-arm experiments.

### Manip Tool Hub

[Manip Tool Hub](https://racingemperor.github.io/manip-tool-hub/) is my public
TypeScript and Next.js project page for exploring embodied intelligence tools.
It presents tool cards, capability categories, benchmark evidence, demos, and
deployment notes in a static GitHub Pages site.

### Tool Validation Workflow

I am interested in benchmark-driven tool validation: finding useful tools,
wrapping them behind consistent interfaces, testing them against public
datasets or deterministic oracles, and promoting only the tools with strong
evidence into an embodied-agent runtime.
